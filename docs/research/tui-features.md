## 1. Screens / Views / Modes

### Top-level surfaces
- **Conversation transcript** (`src/renderer/conversation-*.ts`, `src/core/`) — main chat view. Markdown rendering, syntax highlighting, inline diffs, collapsible blocks (auto-collapse threshold configurable), bookmarks, block copy/save, line-number modes (`off`/`code`/`all`), event navigation by family, markdown tables (tolerant of malformed separators), search overlay, next/prev-error jumps, thinking-strip with live token output.
- **Prompt/composer** — multi-line input with growing height, line-count indicator inside the prompt border, bracketed-paste normalization (pastes >8 lines collapse to a paste marker), kill-ring editing, `@` file picker, `?` help picker, input history + `Ctrl+R` reverse search, prompt undo/redo.
- **Footer / bottom bar** (`src/renderer/bottom-bar.ts`, `footer-tips.ts`) — model name, context bar (full prompt footprint), fresh-input vs cached-context token accounting, tokens/sec (optional), background-process indicator (`process-indicator.ts`), `[Enter]` jump to Fleet.
- **Panel workspace** — split-pane system with panel picker, tabs (`Alt+1..9`), layout control (`/panel open|close|toggle|move|focus|split|width|height`), renderer-owned row budgets, live repaint while idle.

### Six persistent panels (`src/panels/builtin/*.ts`)
| Panel id | File | Purpose |
|---|---|---|
| `fleet` | `builtin/operations.ts`, `fleet-panel.ts` | THE live control room: running agents, workstreams, scheduled jobs; attach/detach (detach never kills), steer, pause/resume, capability-gated interrupt/kill; per-agent transcript (`fleet-transcript.ts`), worktree detail, deep links. Opened by `F2` / `Ctrl+O` / footer Enter. |
| `git` | `builtin/development.ts`, `git-panel.ts` | Git status/log/diff console (via `simple-git`). |
| `diff` | `diff-panel.ts` | Unified diff viewer: session, HEAD, working, staged, or arbitrary git ref. |
| `cost` | `cost-tracker-panel.ts` | Cost tracking console. |
| `tokens` | `builtin/session.ts`, `token-budget-panel.ts` | Token budget/context console (absorbed retired `context` panel via alias). |
| `local-auth` | `local-auth-panel.ts` | Local daemon auth review incl. masked password entry. |

### Config-modal host + ~20 modal surfaces (`src/panels/builtin-modals.ts`, `src/panels/modals/`)
All former heavy panels re-homed as modals on a single "liveness-enforcing config-modal host"; retired panel ids resolve via `registerModalRedirect` (e.g. `/panel open providers` → providers modal). Surfaces: **services**, **subscription**, **remote** (peers/node-hosts), **provider-health** (redirects: `providers`, `accounts`, `provider-health`), **settings-sync**, **local-auth**, **sandbox**, **marketplace**, **plugins**, **skills**, **hooks**, **security**, **policy**, **knowledge**, **memory**, **work-plan**, **keybindings** (redirect `docs`), **pairing** (QR/companion pairing, redirect `qr-code`), **planning** (redirect `project-planning`); retired `sessions` panel folds into the session-picker modal. Modal stack unwinds correctly to the slash-command menu; searchable modals own search/list focus; selection modals use `Space`/`Enter`, `Left`/`Right` for enum/number, `Shift+Left/Right` step-by-10.

### Fullscreen workspaces (`src/renderer/fullscreen-workspace.ts`)
- **Model/provider workspace** (`/model`, `/provider`) — provider-first selection, family/capability/availability/tier filters, synthetic-model grouping ("Top Models" S/A tier vs "All Synthetic", provider counts, S/A/B/C badges), routes for main chat / helper / tool LLM / TTS LLM.
- **Configuration workspace** (`/config`, `/settings`) — all SDK config keys, TTS settings, model routes, secrets-backed settings, MCP trust, subscriptions, feature flags. Categories in `settings-modal-data.ts`: behavior, display, network, storage, tts, ui, web, flags, custom.
- **MCP workspace** (`/mcp` bare) — `src/input/mcp-workspace.ts`.

### Overlays and pickers (`src/renderer/`)
Conversation search overlay (`Ctrl+F`, locked-search `n`/`N` with wrap marker), history search, autocomplete, file picker (`@`), help overlay, model picker overlay, session picker, profile picker, bookmark modal, compaction preview/history/quality views, context inspector (`/context` per-message token breakdown), semantic diff view, selection-modal overlay, panel confirm overlay.

### Onboarding wizard (`src/input/onboarding/onboarding-wizard-steps.ts`)
First-run multi-step flow: loading current settings → **capabilities selection** → **AI provider access** (API-key inventory, OpenAI subscription OAuth start/callback/finish) → **default model** (+ reasoning effort) → **choose external surfaces** (per-surface setup fields, auto-start, secret policy) → **access and accounts** → **shell experience** (HITL, guidance, permissions) → **network** (mode local/shared, IPs, service/browser/webhook ports) → Cloudflare step → daemon-adopt step. Also `goodvibes onboarding status` CLI.

### Other modes
- **Alternate screen buffer** raw ANSI; `--no-alt-screen` for logging terminals.
- **Non-interactive print mode**: `goodvibes run|exec [prompt]` with `--output text|json|stream-json`, `--print`, `--prompt/-p` prompt seeding.
- **Browser operator surface** (`web.enabled`, port 3423) — daemon-hosted operator shell at `/api/control-plane/web`.
- **Crash-recovery prompt** on startup (JSONL snapshots + fsync append-only journal replay; `docs/session-durability.md`).

## 4. Session / Chat Model

- **Turn lifecycle**: explicit submit/steer/follow-up semantics with lifecycle-state tracking, correlation/causation IDs, return-context summaries (`behavior.returnContextMode` off/local/assisted), knowledge capture, compaction (hybrid structured v2 with preview/history/quality UI), guidance, retention, idempotency keys (dedupe across replay/reconnect/retry). Terminal events: `TURN_COMPLETED`, `TURN_ERROR`, `TURN_CANCEL`, `PREFLIGHT_FAIL`; `STREAM_END` is per-provider-stream, non-terminal.
- **Providers** (per `package.json` deps + README): native `openai`, `anthropic` (`@anthropic-ai/sdk`), `openai-codex`, `gemini`, `amazon-bedrock` (`@anthropic-ai/bedrock-sdk` + `@aws/bedrock-token-generator`), `amazon-bedrock-mantle`, `anthropic-vertex` (`@anthropic-ai/vertex-sdk`), `github-copilot`, `inceptionlabs`; ~25 OpenAI-compat/gateway providers (openrouter, groq, cerebras, mistral, deepseek, together, xai, litellm, cloudflare-ai-gateway, vercel-ai-gateway, etc.); alias normalization; custom provider JSON (hot-reloaded) in `~/.goodvibes/tui/providers/*.json`; local-server auto-discovery (Ollama, LM Studio, vLLM, llama.cpp/LocalAI, TGI, Jan, GPT4All, KoboldCpp, Aphrodite). Model catalog from models.dev (4,000+ models, 24h TTL) with benchmark tiers.
- **Synthetic failover provider**: same model grouped across backends; 429 → immediate next-provider retry; 5xx/network → 5s cooldown retry; 400 no failover; ≤120s cooldowns auto-waited; cross-MODEL cascade for free tier only (benchmark-ranked, inline notify); paid/subscription never auto-switch models — user gets recovery options. Reasoning effort levels instant/low/medium/high.
- **Sessions**: save/load/rename/resume/fork/search/delete/export; conversation branches (`/branch`, `/fork`, `/merge`); CLI `--continue/--resume/--fork`; crash recovery via JSONL snapshots + fsync-per-record append-only journal replayed at every resume seam (SIGKILL-proof); workspace checkpoints separate from conversation (`/checkpoint`, `/rewind` restores files only).
- **Agents**: in-process subagents, isolated history, scoped tool registries, optional git worktree per agent (merge-back on completion, cleanup on cancel), ACP (`@agentclientprotocol/sdk` stdio ndJsonStream) communication. Built-in archetypes engineer/reviewer/tester/researcher/general; custom markdown archetypes in `.goodvibes/agents/*.md` (YAML frontmatter: name/description/tools/model; body = system prompt). `agent` tool: 15 modes (spawn, batch-spawn, status, cancel, list, get, wait, message, budget, plan, wrfc-chains, wrfc-history, cohort-status, cohort-report). Guards: `danger.agentRecursion` (default false), `danger.maxGlobalAgents` (8). WRFC (write-review-fix-check) chains with constraint tracking (`c:N/M` badges, `[SAT]/[UNS CRIT|MAJOR|MINOR]/[UNV]`, inherited-constraint markers, controller-flagged synthetic issues). Multi-phase workstreams (`/workstream`); helper-model routing for grunt work (`helper.*`).
- **Permissions/approvals**: modes `prompt` (default; asks before write/edit/exec/fetch/agent/workflow/MCP), `allow-all`, `custom` (`permissions.tools.<name>` = allow/prompt/deny). Layered policy evaluation (prefix rules, arg-shape, path scope, network scope, mode constraints), decision logs, policy bundles with signing/verification, preflight review, divergence simulation, promote/rollback/diff, rule suggestions generated from actual approval decisions. `/approval` front door; approvals visible in channel status.
- **Prompt caching**: `cache.enabled` (default true), `cache.stableTtl` (1h for system prompt + tools), hit-rate monitoring with warnings.
